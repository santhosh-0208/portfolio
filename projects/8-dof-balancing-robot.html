8-DOF Self-Balancing Robot

Control Systems Whole-Body Control MuJoCo Sim Reinforcement Learning

Project Overview

The self-balancing robot was where I learned the fundamentals of control systems. This project is my attempt to push those ideas further.

The goal is to build a dynamically balancing mobile manipulator with eight degrees of freedom, capable of maintaining balance while performing whole-body motions. Unlike the original two-wheeled robot, balancing is no longer the only objective. The robot must simultaneously regulate body height, generate translational motion, and coordinate multiple joints while maintaining stability.

This project serves as a platform for exploring whole-body control, model-based control methods, and reinforcement learning on a physical robotic system.

Current Progress

The robot is currently capable of:

  • Maintaining dynamic balance on two wheels
  • Controlling forward and backward translational velocity
  • Regulating body height through coordinated squatting motions
  • Executing multiple control objectives simultaneously

At the current stage, the robot can independently control its center of mass height and translational velocity while maintaining balance.

Mechanical Design

The robot consists of:

  • Two drive wheels providing actuation and balance control
  • Four articulated legs
  • Eight total degrees of freedom
  • Inertial sensing for attitude estimation
  • Embedded real-time control hardware

The leg mechanisms allow the robot to actively reposition its center of mass and modify its body height, introducing a richer set of behaviors compared to a traditional inverted pendulum.

Control Architecture

The current controller is built as a hierarchy of control loops, each managing different aspects of the robot's behavior:

Balance Control

The lowest level controller stabilizes the robot as an inverted pendulum by regulating body pitch. This provides the fundamental stability required for all other operations.

Velocity Control

A higher-level loop commands translational motion while preserving stability. This allows the robot to move forward and backward while maintaining its balance.

Height Regulation

The leg joints coordinate to adjust body height through squatting motions while maintaining the desired center of mass trajectory.

Managing the interactions between these objectives has been one of the most challenging aspects of the project. Changes in body height affect the system dynamics, which in turn influence balancing behavior and velocity tracking.

Key Challenges

The most interesting challenge so far has been dealing with the coupling between different control objectives.

Unlike the simpler balancing robot, where the plant dynamics remain relatively fixed, this robot continuously changes its own dynamics through leg motion. A controller that works well at one body height may behave differently at another.

This has forced me to think much more carefully about:

  • Dynamic Stability - How changing body height affects stability margins
  • State Estimation - Accurately tracking position and velocity with changing dynamics
  • Controller Interactions - How gains at one level affect others
  • Gain Scheduling - Adapting controller parameters based on operating point
  • Whole-Body Coordination - Ensuring all joints work together toward common goals

Many of the problems have not been software bugs but systems-level problems where the robot behaves exactly as instructed, yet not as intended. This teaches a deeper lesson about the difference between correct code and correct behavior.

Next Steps: Adaptive Control with Reinforcement Learning

The current system relies on manually tuned PID controllers. This works well for nominal operating conditions but struggles with variations in the environment or changes in the system dynamics.

The next stage of the project is to investigate whether reinforcement learning can improve performance by dynamically adapting controller parameters based on the robot's operating conditions.

Rather than replacing the existing control architecture, the goal is to use learning as an augmentation layer on top of proven classical control methods. This approach combines:

  • The robustness and interpretability of PID control
  • The adaptability of modern learning-based techniques

The result should be a system that maintains the reliability of classical control while gaining the flexibility of adaptive learning.
