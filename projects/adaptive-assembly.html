Adaptive Robotic Assembly

The Challenge

This project started from a question I kept running into while working with industrial robots: how do you make a robot reliably complete an assembly task when the world is never exactly as expected?

Traditional industrial automation works brilliantly when everything is fixtured and repeatable. But even small position errors can cause insertion tasks to fail. Humans naturally compensate using vision and touch; robots need to learn to do the same.

Problem Statement

Consider a simple peg-in-hole insertion task:

  • The peg's position might be off by a few millimeters
  • The hole's orientation might be slightly misaligned
  • Contact forces provide rich information about misalignment, but only after contact
  • Vision can guide the approach, but it's noisy and delayed

The robot must fuse information from multiple sources and adapt its behavior in real-time.

Solution Architecture

I wanted to explore how vision and force sensing could be combined to create more robust assembly behavior. The system consisted of three main components:

1. Image-Based Visual Servoing

A computer vision pipeline continuously estimated alignment errors using camera feedback. This provided global information about the peg's position relative to the hole before contact.

2. Virtual Force-Torque Sensor

A virtual force-torque sensor estimated interaction forces from joint torque sensor measurements. This provided immediate, high-bandwidth feedback about contact dynamics.

3. Sensor Fusion Layer

A sensor fusion layer combined both sources of information to generate a more reliable estimate of the assembly state and guide the insertion motion.

Key Technical Challenges

Vision-Force Tradeoffs

One of the biggest lessons from this project was understanding the tradeoffs between sensing modalities:

  • Vision: Provides rich information but can be noisy, occluded, and delayed
  • Force: Immediate and highly informative during contact but tells you very little before contact occurs

Timing and Confidence

Getting both systems to complement each other required thinking carefully about:

  • Which source to trust at different stages of the insertion
  • How to weight conflicting information
  • When to switch between vision-guided and force-guided control
  • How to detect and recover from failures

The Broader Lesson

This project also introduced me to a recurring theme in robotics: the best solution is often not choosing between two approaches, but finding a way for them to work together.

Rather than asking "Should I use vision or force control?", the right question is "How can vision and force control complement each other?" Each has its strengths, and the magic happens when they're orchestrated properly.

Impact and Reflection

Looking back, this project was my first serious exploration into the intersection of perception, control, and robot learning. It pushed me to think beyond individual algorithms and start designing complete robotic systems.

The skills developed here—sensor integration, real-time control, handling uncertainty, and multi-modal reasoning—have become foundational to how I approach robot systems design.
